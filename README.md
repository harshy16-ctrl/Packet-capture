# Packet-capture
Cyber Security Internship – Task 5: Capture and Analyze Network Traffic Using Wireshark

Author: Harsh Yadav

Objective

To capture live network packets using Wireshark and analyze different
network protocols to understand how devices communicate over a network.

Tools Used

-   Wireshark
-   Web Browser
-   Command Prompt / Terminal
-   Internet Connection

Steps Performed

1.  Installed Wireshark.
2.  Started packet capture on the active network interface.
3.  Generated traffic by browsing websites and using ping commands.
4.  Stopped capture after approximately one minute.
5.  Applied filters: dns, tcp, http, udp.
6.  Identified multiple protocols.
7.  Exported the capture as network_capture.pcap.

Findings

Protocols Identified: - DNS - TCP - HTTP/HTTPS - UDP

Outcome

Successfully captured and analyzed network traffic using Wireshark and
learned packet analysis, protocol identification, and troubleshooting
basics.

Interview Questions and Answers

1.  What is Wireshark used for? Wireshark is a network protocol analyzer
    used to capture and inspect network traffic.

2.  What is a packet? A packet is a small unit of data transmitted over
    a network.

3.  How do you filter packets in Wireshark? Using display filters such
    as dns, tcp, http, and udp.

4.  What is the difference between TCP and UDP? TCP is reliable and
    connection-oriented, while UDP is faster and connectionless.

5.  What is a DNS query packet? A packet sent to a DNS server to resolve
    a domain name into an IP address.

6.  How can packet capture help in troubleshooting? It helps identify
    connectivity issues, packet loss, and protocol errors.

7.  What is a protocol? A set of rules that governs communication
    between network devices.

8.  Can Wireshark decrypt encrypted traffic? In some cases, if the
    required decryption keys are available.

Key Concepts: - Packet Capture - Protocol Analysis - TCP/IP - DNS -
HTTP/HTTPS - Network Troubleshooting - Packet Filtering
